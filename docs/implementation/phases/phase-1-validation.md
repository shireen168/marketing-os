# Phase 1: Research (Day 1-2)

**Goal:** Prove the problem is real. Talk to actual users. Validate demand before building anything.

**Skills:** 7  
**Outcome:** MVP scope locked, assumptions validated  
**End checkpoint:** `/context-save` (saves all validation decisions)

---

## Skill 1: `/office-hours`

**Brainstorm + validate the sleep tracking idea**
- Get hard questions about demand (Is this real? Who wants it? What would they pay?)
- Explore builder mode (you're testing an idea, not validating a company)
- Output: Design doc with problem statement + premises

**Next:** Skill 2 (`/investigate`)

---

## Skill 2: `/investigate`

**Research sleep tech landscape. Root cause analysis on why this problem exists.**
- Deep-dive on existing sleep trackers (Oura, Apple Watch, Whoop)
- Find the core failure point (why do users drop off?)
- Validate the problem is real, not hypothetical
- Output: Research findings on competitor failures + user pain

**Next:** Skill 3 (`/learn`)

---

## Skill 3: `/learn`

**Deep learning on sleep science + wearable hardware constraints**
- Sleep fundamentals: REM vs NREM, sleep cycles, actigraphy
- Hardware constraints: battery vs accuracy tradeoffs, sensor types
- Understand what data you can actually collect
- Output: Knowledge base on sleep biology + architecture constraints

**Next:** Skill 4 (`/customer-research`)

---

## Skill 4: `/customer-research`

**Talk to actual users. Confirm demand signals.**
- Target: Sleep-conscious professionals (health trackers, biohackers)
- Ask: "What's the #1 thing you'd change about your current sleep tracker?"
- Validate: Would they pay? How much? How often do they use the current solution?
- Output: User quotes + demand signals (real behavior, not interest)

**Next:** Skill 5 (`/competitor-profiling`)

---

## Skill 5: `/competitor-profiling`

**Map competitive landscape. What are incumbents doing well/poorly?**
- Analyze: Oura Ring, Apple Watch, Whoop, Fitbit, legacy sleep apps
- Focus: Accuracy claims, retention rates, pricing models, positioning
- Identify: What haven't competitors solved? Where's your wedge?
- Output: Competitor matrix (features, pricing, weaknesses)

**Next:** Skill 6 (`/marketing-psychology`)

---

## Skill 6: `/marketing-psychology`

**Understand deep motivation. Why do people actually track sleep?**
- Angles: Health anxiety, productivity optimization, quantified self, biohacking
- Find: Which motivation resonates most with your customers?
- Understand: What triggers action? What triggers abandonment?
- Output: Psychological positioning frameworks + messaging angles

**Next:** Skill 7 (`/pair-agent`)

---

## Skill 7: `/pair-agent`

**Collaborate with Claude. Narrow scope to MVP.**
- Question: "Given everything we've learned, what's the smallest version someone pays for?"
- Goal: 1 core feature, not 10 features
- Outcome: Locked MVP definition (e.g., "Sleep duration + REM% + 7-day battery")
- Output: MVP scope document
