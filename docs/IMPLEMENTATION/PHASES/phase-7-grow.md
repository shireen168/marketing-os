## Skill 56: `/churn-prevention`

**Identify churn points. Why users drop off.**
- Analyze: At what point do users stop tracking? Day 3? Day 7? Day 30?
- Root cause: Too complicated? Didn't see value? Battery drain annoyed them?
- Fix: If churn at day 7, send "Here's what your sleep pattern says" email
- Experiment: Test fixes and measure impact on retention
- Output: Churn analysis + retention experiments
